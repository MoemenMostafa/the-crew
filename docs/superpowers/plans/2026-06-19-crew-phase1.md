# The Crew — Phase 1 (Foundation + Adam end-to-end) Implementation Plan

> **For agentic workers:** Use superpowers:subagent-driven-development or superpowers:executing-plans to implement task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** A config-driven, always-on Python service that runs one persona (Adam) as a Claude Agent SDK session over Slack Socket Mode, operating the real Loquina repo with full-hands-on guardrails, audit logging, editable personality, and persistent memory — built so adding Eva/Zakarya/Sara (Phase 2) is configuration, not new code.

**Architecture:** One async process. `crew.yaml` + per-persona dirs define everything. A generic `Persona` is loaded from files; an `AgentSession` wraps `claude_agent_sdk` (resumable, memory-injected, guardrail-hooked); a `SlackConnector` runs one Socket Mode app per persona and routes events to a `Router` that dispatches turns onto per-persona queues. Slack is the inter-agent bus (Phase 2). Phase 1 ships the whole spine with Adam only.

**Tech Stack:** Python 3.10 (`python3.10`, own venv) · `claude-agent-sdk` (drives the installed authenticated `claude` CLI 2.1.183) · `slack-bolt` (Socket Mode) · `PyYAML` · `pytest`/`pytest-asyncio`.

## Global Constraints

- Python interpreter: `python3.10` (3.10.10). Build a venv at `crew/.venv`; never use system 3.9.
- Default model: `claude-opus-4-8` (per persona, overridable in `persona.yaml`).
- Loquina repo path: `/Users/m.mostafa/Workspace/code/plauda`; protected branch: `main`.
- No secrets in git: tokens live in `crew/.env` (gitignored); `.env.example` documents them.
- Engine auth: the Agent SDK uses the local `claude` CLI's credentials — no `ANTHROPIC_API_KEY` required.
- All agent tool actions are audit-logged to `crew/.logs/audit.jsonl`.
- Tests must pass without live Slack or live Claude (mock those boundaries).
